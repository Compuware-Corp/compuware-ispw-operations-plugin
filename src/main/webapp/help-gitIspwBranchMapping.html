• The 1st field is GIT to ISPW branch mapping, support wild card, for example, **/dev1 => DEV1, **/dev2 => DEV2 etc.
  • The 2nd field can be per-commit (container will be created for each commit), per-branch (container will be re-used per branch commit) or custom (user can specify a container name as description, user must specify 3rd field).
  • The 3rd field need to be specified as container description only if user choose 'custom' in the 2nd field.
